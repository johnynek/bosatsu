BosatsuUI Bindings (per-item, O(1) updates):
["todos", "{id}", "text"] -> #todo-{id}-text.textContent
["todos", "{id}", "completed"] -> #todo-{id}-checkbox.className
["stats", "total"] -> #total-count.textContent
["stats", "active"] -> #active-count.textContent
["stats", "completed"] -> #completed-count.textContent

Why This Matters for Performance

When you toggle a todo, BosatsuUI:

  1. Looks up bindings for ["todos", "3", "completed"]
  2. Directly updates #todo-3-checkbox.className
  3. Also updates stat counters via their bindings

React/Elm would re-render the entire list and diff to find what changed. With 100 todos, BosatsuUI is still O(1) while VDOM is O(100).
