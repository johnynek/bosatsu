Why Randomness Matters

This benchmark uses a seeded random number generator to create unpredictable particle physics. Each mouse movement triggers chaos calculations that can't be optimized away at compile time. Both frameworks receive identical random inputs.

BosatsuUI approach: Each particle has a direct binding like ['particles', '42', 'x'] → #particle-42.style.left. Updating one particle is O(1) — just a map lookup and direct DOM write.

React approach: Every mouse move triggers setState for all particles. React must create a new VDOM tree, diff all particles, and reconcile changes — O(n) per frame.

The chaos seed ensures reproducible "randomness" — same seed, same particle behavior. Increase physics noise to add more chaotic movement that prevents any caching.
